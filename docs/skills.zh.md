## 🎯 核心技能

### `/brainstorm` - 头脑风暴

**目的**: 结构化产品构思

**何时使用**:
- 开始新项目
- 探索功能想法
- 重新定义问题

**流程**:
1. 5W1H 分析
2. 问题重构
3. 解决方案替代方案
4. 工作量估算

**输出**: 设计文档（输入到下游技能）

---

### `/ceo-review` - CEO 审视

**目的**: 战略对齐和范围控制

**何时使用**:
- 重大功能前
- 范围蔓延担忧
- 资源分配决策

**模式**:
1. **扩展** - 增加范围以获得更大影响
2. **选择性扩展** - 扩展特定领域
3. **保持范围** - 保持当前范围
4. **缩减** - 减少范围以更快发布

**输出**: 已批准的范围文档

---

### `/arch-design` - 架构设计

**目的**: 技术架构和数据流设计

**何时使用**:
- 新系统设计
- 重大重构
- 集成规划

**交付物**:
- 数据流图
- API 规范
- 状态机
- 错误处理策略
- 测试矩阵

---

### `/code-review` - 代码审查

**目的**: Staff Engineer 级别代码审查

**何时使用**:
- 合并 PR 前
- 重构验证
- 安全审计

**检查项**:
- 逻辑正确性
- 边界情况
- 性能瓶颈
- 安全漏洞
- 代码风格

**自动修复**:
- 风格问题
- 简单错误
- 文档缺失

---

### `/qa-test` - 质量测试

**目的**: 使用真实浏览器自动化测试

**何时使用**:
- 发布前
- 错误验证
- 回归测试

**能力**:
- 真实浏览器自动化
- 截图对比
- 流程验证
- 错误自动修复

**输出**:
- 测试报告
- 回归测试
- 错误修复（如果启用）

---

### `/security` - 安全审计

**目的**: OWASP + 等保合规审计

**何时使用**:
- 生产环境前
- 季度审计
- 重大变更后

**标准**:
- OWASP 十大漏洞
- 等保 2.0 三级
- STRIDE 威胁模型

**输出**:
- 安全报告
- 风险评级
- 修复计划

---

### `/ship` - 一键发布

**目的**: 测试 → PR → 合并一键完成

**何时使用**:
- 功能完成
- 错误修复就绪
- 热修复部署

**流程**:
1. 同步主分支
2. 运行测试
3. 审计覆盖率
4. 创建 PR
5. 推送到远程

**输出**:
- PR 链接
- 测试结果
- 覆盖率报告

---

### `/deploy` - 部署验证

**目的**: CI/CD 监控和验证

**何时使用**:
- PR 合并后
- 生产环境部署
- 回滚验证

**流程**:
1. 等待 CI
2. 监控部署
3. 验证健康检查
4. 确认生产就绪

## 🛡️ 安全技能

### `/careful` - 谨慎模式

**目的**: 破坏性操作前警告

**触发**:
- `rm -rf` / 删除文件
- `DROP TABLE` / 删除表
- `git push --force` / 强制推送
- 数据库迁移

**使用**:
```
/careful
# 然后继续操作
```

---

### `/freeze` - 冻结模式

**目的**: 锁定编辑到特定目录

**使用**:
```
/freeze ./src/components
# 只允许编辑 ./src/components 内的文件
```

---

### `/guard` - 守护模式

**目的**: 完整安全模式（谨慎+冻结）

**使用**:
```
/guard ./critical-code
# 生产环境工作的最大安全性
```

## 📝 技能使用模式

### 模式 1: 新功能开发

```
/brainstorm        → 设计文档
/ceo-review        → 范围批准
/arch-design       → 技术规范
[实现代码]
/code-review       → 代码验证
/qa-test           → 测试
/ship              → 发布 PR
```

### 模式 2: 错误修复

```
/investigate       → 根因分析
[修复代码]
/code-review       → 修复验证
/qa-test           → 回归测试
/ship              → 热修复 PR
```

### 模式 3: 安全补丁

```
/guard             → 开启安全模式
/security          → 漏洞扫描
[修复漏洞]
/code-review       → 安全审查
/qa-test           → 测试
/ship              → 安全 PR
```

## 🔧 自定义

### 添加自定义技能

1. 创建技能目录:
   ```
   skills/my-skill/
   ├── SKILL.md
   └── script.js
   ```

2. 定义技能清单:
   ```yaml
   name: my-skill
   command: /my-skill
   description: "我的自定义技能"
   ```

3. 在配置中注册:
   ```yaml
   skills:
     custom:
       - my-skill
   ```
