## 🤝 How to Contribute / 如何贡献

### 1. Report Issues / 报告问题

Found a bug? Have a suggestion? / 发现错误？有建议？

- **GitHub Issues**: [github.com/OasysCore/DragonStack-/issues](https://github.com/OasysCore/DragonStack-/issues)
- **Bug Reports**: Use the bug report template / 使用错误报告模板
- **Feature Requests**: Use the feature request template / 使用功能请求模板

### 2. Submit Pull Requests / 提交 PR

Want to contribute code? / 想贡献代码？

1. **Fork** the repository / Fork 仓库
2. **Create** a feature branch / 创建功能分支
3. **Make** your changes / 做出你的修改
4. **Test** your changes / 测试你的修改
5. **Submit** a pull request / 提交 PR

### 3. Improve Documentation / 改进文档

Documentation is code. / 文档就是代码。

- Fix typos / 修复拼写错误
- Add examples / 添加示例
- Translate content / 翻译内容
- Clarify instructions / 澄清说明

### 4. Share Your Experience / 分享你的经验

- Write blog posts / 写博客文章
- Create video tutorials / 创建视频教程
- Share on social media / 在社交媒体上分享
- Present at meetups / 在聚会上演讲

## 📝 Contribution Guidelines / 贡献指南

### Code Style / 代码风格

- Follow existing patterns / 遵循现有模式
- Add comments in EN + CN / 添加中英文注释
- Write tests for new features / 为新功能编写测试
- Update documentation / 更新文档

### Commit Messages / 提交信息

```
type(scope): subject in English

Description in Chinese / 中文描述

Fixes #123
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### Pull Request Process / PR 流程

1. **Describe** what changed and why / 描述什么改变了以及为什么
2. **Reference** related issues / 引用相关问题
3. **Ensure** all tests pass / 确保所有测试通过
4. **Request** review from maintainers / 请求维护者审查

## 🏗️ Development Setup / 开发设置

```bash
# Clone your fork / 克隆你的 fork
git clone https://github.com/YOUR_USERNAME/DragonStack-.git
cd DragonStack-

# Install dependencies / 安装依赖
npm install

# Run tests / 运行测试
npm test

# Start development / 开始开发
npm run dev
```
