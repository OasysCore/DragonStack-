# 龙栈设计理念 / DragonStack Philosophy

> 工具背后的思考  
> The thinking behind the tools  
> Last Updated: 2026-03-23

---

## 🎯 核心信念

### 1. 流程胜于工具

工具是手段，不是目的。好的流程配上简单的工具，胜过没有流程的顶级工具。

**龙栈首先是流程，其次才是工具集。**

---

### 2. 一人，二十专家

现代 AI 使单个开发者能够以完整工程团队的能力运作：

- **CEO** - 战略方向
- **产品经理** - 功能优先级
- **架构师** - 系统设计
- **高级工程师** - 代码质量
- **QA 负责人** - 测试验证
- **安全官** - 威胁评估
- **DevOps** - 部署监控

**龙栈为这个虚拟团队提供运作结构。**

---

### 3. 默认安全

破坏性操作应该需要明确的意图：

- `/careful` - 危险前警告
- `/freeze` - 限制范围
- `/guard` - 最大安全

**安全不是可选项。它内置于每个工作流。**

---

### 4. 双语设计

龙栈首先服务中国开发者：

- 所有文档中英文
- 针对国内平台优化
- 尊重中国网络现实

**技术应该服务于用户的上下文。**

## 🛡️ 安全原则

### 三守护

1. **Careful** - 质疑破坏性操作
2. **Freeze** - 限制编辑范围
3. **Guard** - 结合两者获得最大安全

### 何时使用安全模式

| 场景 | 推荐守护 |
|-----|---------|
| 生产热修复 | `/guard` |
| 数据库迁移 | `/careful` |
| 重构 | `/freeze ./target-dir` |
| 新功能 | 无（快速迭代）|
| 学习 | 无（自由实验）|

## 🌏 平台理念

### 为什么 Trae 作为默认？

1. **可访问性** - 无需代理
2. **成本** - 完全免费
3. **能力** - Claude 3.5 是世界级
4. **中国上下文** - 为中国开发者构建

### 何时使用 Cursor？

1. **高级功能** - 需要 Composer 模式
2. **预算可用** - $20/月可接受
3. **代理可用** - 稳定 VPN 访问
4. **最大生产力** - 每秒都很重要

## 📚 从 gstack 学习

龙栈受 [gstack](https://github.com/garrytan/gstack) 启发，但在关键方面有所不同：

| 方面 | gstack | DragonStack |
|-----|--------|-------------|
| **主要平台** | Claude Code | Trae / Cursor |
| **目标用户** | 全球 | 中国开发者 |
| **网络** | 假设全球互联网 | 在 GFW 后工作 |
| **语言** | 英文 | 双语（EN+CN）|
| **成本** | 免费（需要 Claude）| 免费（Trae 免费）|

**我们站在巨人的肩膀上，但为我们的上下文构建。**
