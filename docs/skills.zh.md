## 🎯 核心技能

### `/brainstorm` - 头脑风暴

**目的**: 结构化产品构思

**何时使用**:
- 开始新项目
- 探索功能想法
- 重新定义问题

**流程**:
1. 5W1H 分析
2. 问题重构
3. 解决方案替代方案
4. 工作量估算

**输出**: 设计文档（输入到下游技能）

---

### `/code-review` - 代码审查

**目的**: Staff Engineer 级别代码审查

**何时使用**:
- 合并 PR 前
- 重构验证
- 安全审计

**检查项**:
- 逻辑正确性
- 边界情况
- 性能瓶颈
- 安全漏洞
- 代码风格

**自动修复**:
- 风格问题
- 简单错误
- 文档缺失

---

### `/ship` - 一键发布

**目的**: 测试 → PR → 合并一键完成

**何时使用**:
- 功能完成
- 错误修复就绪
- 热修复部署

**流程**:
1. 同步主分支
2. 运行测试
3. 审计覆盖率
4. 创建 PR
5. 推送到远程

**输出**:
- PR 链接
- 测试结果
- 覆盖率报告

---

## 🛡️ 安全技能

### `/careful` - 谨慎模式

**目的**: 破坏性操作前警告

**触发条件**:
- `rm -rf` 删除文件
- `DROP TABLE` 删除表
- `git push --force` 强制推送
- 数据库迁移

### `/freeze` - 冻结模式

**目的**: 锁定编辑到特定目录

### `/guard` - 守护模式

**目的**: 完整安全模式（谨慎+冻结）

## 📝 技能使用模式

### 模式 1: 新功能开发
```
/brainstorm     → 设计文档
[实现代码]
/code-review    → 代码验证
/ship           → 发布 PR
```

### 模式 2: 错误修复
```
/brainstorm     → 根因分析
[修复代码]
/code-review    → 修复验证
/ship           → 热修复 PR
```

## 🔧 自定义

### 添加自定义技能

1. 创建技能目录:
   ```
   skills/my-skill/
   ├── SKILL.md
   └── script.js
   ```

2. 定义技能清单:
   ```yaml
   name: my-skill
   command: /my-skill
   description: "我的自定义技能"
   ```

3. 在配置中注册:
   ```yaml
   skills:
     custom:
       - my-skill
   ```
